# inspiration-deep-discussion - 深度讨论 Skill

## 技能定位

**用途**：与用户进行深度话题讨论，而不是单向输出答案或记录灵感

**触发场景**：
- 用户说"想讨论一下..."、"聊聊..."、"帮我分析一下..."
- 用户明确表示"不是记录灵感"、"不要直接回答"
- 话题复杂，需要多轮对话才能理清

**与 inspiration-record 的区别**：
| 维度 | inspiration-record | inspiration-deep-discussion |
|------|-------------------|----------------------------|
| 目的 | 快速记录想法 | 深度探讨问题 |
| 输出 | 简短 MD 文件 | 多轮对话 + 讨论记录 |
| 互动 | 单向记录 | 双向提问讨论 |
| 存储 | `memory/inbox/` | `memory/discussions/` |

---

## 核心方法论

### 1. 沟通原则（强制遵守）

**每次只问 1-2 个问题**：
- ❌ 不要一次性问 5-6 个问题（像审问）
- ✅ 每次 1 个核心问题，最多 2 个
- ✅ 等用户回答后再继续追问

**跳出用户思维框架提问**：
- ❌ 不要只在用户的问题范围内展开
- ✅ 引入外部视角、新角度、用户可能没想到的点
- ✅ 例如："你考虑的线下店，到底是为了什么目的？"（销售/展示/服务/渠道）

**主动搜索已有灵感**：
- ❌ 不要把每个话题当成孤立的
- ✅ 搜索 `memory/` 目录，寻找关联的已有灵感
- ✅ 例如：讨论线下零售 → 关联到"C 端产品小 B 化"灵感

**长期讨论存储**：
- ❌ 不要依赖会话上下文（会溢出）
- ✅ 使用 `memory/discussions/` 目录存储讨论记录
- ✅ 每轮对话后及时更新记录

**回归第一性原理**：
- ❌ 不要在表层问题上打转（如"开在哪里"、"怎么引流"）
- ✅ 持续追问最原点的驱动力（如"用户为什么买 IT 产品"）
- ✅ 找到问题的根源，而不是在症状上修修补补
- ✅ 当讨论陷入僵局时，主动引导回到第一性原理

---

### 2. 讨论流程

#### 🔄 阶段的灵活运用

**6 个阶段不是线性流程，而是工具箱**：

- ✅ **可以跳跃**：用户突然提到新信息 → 跳回"了解背景"阶段
- ✅ **可以重复**：某个阶段没聊透 → 重新回到该阶段
- ✅ **可以跳过**：某些话题不需要全部 6 个阶段
- ✅ **可以并行**：多个阶段的问题可以交织进行

**判断何时切换阶段的信号**：

| 信号 | 应对 |
|------|------|
| 用户说"其实我还没说清楚背景..." | 跳回第一阶段 |
| 用户回答很模糊 | 重复第三阶段（深入场景） |
| 用户已经有清晰目标 | 跳过第二阶段，直接进入第三阶段 |
| 讨论陷入僵局 | 进入第四阶段（引入新视角） |
| 用户提到"之前说过..." | 进入第五阶段（关联已有思考） |

**核心原则**：阶段是服务于讨论质量的，不是必须遵守的流程。

---

#### 第一阶段：了解背景（1-2 轮）

**目标**：搞清楚用户为什么问这个问题

**典型问题**：
- "你问这个问题，是纯好奇，还是跟工作/投资/创业有关？"
- "能说说你的背景吗？这样我们聊的时候能更有针对性"

**记录要点**：
- 用户的职业/身份
- 业务模式
- 核心困惑

---

#### 第二阶段：明确目的（1-2 轮）

**目标**：搞清楚用户想要什么

**典型问题**：
- "你心里想的线下店，主要是为了什么目的？"
- "你是想帮助 B 端客户，还是自己做 C 端零售？"

**记录要点**：
- 用户的目标（可能有多个）
- B 端思维 vs C 端思维

---

#### 第三阶段：深入场景（2-3 轮）

**目标**：了解具体困境和痛点

**典型问题**：
- "你的下游代理商现在面临的最大困难是什么？"
- "是他们主动抱怨过什么，还是你观察到的？"

**记录要点**：
- 具体场景（如三种店型）
- 共同痛点 vs 差异化问题

---

#### 第四阶段：引入新视角（2-3 轮）

**目标**：帮用户跳出当前框架

**典型问题**：
- "你觉得线下店一定要靠卖货的差价来赚钱吗？"
- "有没有见过谁是用非差价的方式赚钱的？"

**记录要点**：
- 用户尝试过的方案
- 失败原因分析

---

#### 第五阶段：关联已有思考（1-2 轮）

**目标**：建立跨话题的关联

**动作**：
- 搜索 `memory/inbox/` 和 `memory/reports/`
- 寻找相关的已有灵感
- 例如："之前讨论过'C 端产品小 B 化'，你还记得吗？"

**记录要点**：
- 关联的灵感主题
- 关联性说明

---

#### 第六阶段：挖掘隐性需求（2-3 轮）

**目标**：找到用户和消费者都不知道的隐性需求

**典型问题**：
- "你买过什么产品，是买之前不知道需要，但用了之后觉得真香的？"
- "你的最终用户在买完之后遇到过什么麻烦？"

**记录要点**：
- 隐性需求的特征
- 可能的服务方向

## 搜索与关联机制

### 搜索时机

**何时搜索**：
- 用户提到"之前讨论过..."时
- 话题明显与已有灵感相关时
- 讨论陷入僵局，需要新视角时

**搜索关键词**：
- 从用户当前话题提取 2-3 个核心词
- 例如："线下零售" → 搜索"零售"、"C 端"、"销售模式"

**搜索范围**：
- `memory/inbox/`（原始灵感）
- `memory/reports/`（研究报告）
- `memory/discussions/`（历史讨论）

---

### 关联方式

**在讨论中引入**：
> "我记得之前讨论过'[灵感主题]'，你还记得吗？那个话题和现在这个话题有关联..."

**在文档中记录**：
```markdown
## 🔗 关联的已有灵感

| 灵感主题 | 时间 | 关联性 |
|---------|------|--------|
| C 端产品小 B 化 | 2026-02-26 | 都涉及给个人用户提供 B 端式服务 |
```

## 上下文管理

### 溢出预防

**监控上下文用量**：
- 每次回复末尾标注：`💬 [已用]/[上限] [百分比]`
- 当用量 > 50% 时，主动提醒用户

**溢出处理**：
1. 提醒用户："上下文已用了 X%，我们可能需要保存一下讨论进度"
2. 创建/更新 `memory/discussions/` 文件
3. 总结当前讨论要点
4. 从文件中读取继续讨论

---

### 文件更新频率

**每轮对话后**：
- 更新 `memory/discussions/` 文件
- 添加新一轮的对话记录
- 标注时间和关键洞察

**示例**：
```markdown
### 第七轮（15:59）
**用户**: 最大障碍是供需双方都不知道需要什么服务

**助理**: 问了一个关于隐性需求的问题

**核心困境**：隐性需求，无法被清晰表达
```

## 质量要求

### 提问质量

- ✅ 每次只问 1-2 个问题
- ✅ 问题能帮用户跳出思维框架
- ✅ 问题基于用户的回答，不是预设的
- ❌ 不要一次性问 5-6 个问题
- ❌ 不要像审问一样连续追问

### 关联质量

- ✅ 主动搜索已有灵感
- ✅ 明确说明关联性
- ✅ 帮用户建立跨话题的思考
- ❌ 不要强行关联
- ❌ 不要把不相关的灵感扯进来

### 记录质量

- ✅ 每轮对话后及时更新
- ✅ 记录用户的核心观点
- ✅ 记录讨论的关键洞察
- ❌ 不要只记录助理的输出
- ❌ 不要遗漏用户的重要反馈

## 示例对话

### 场景：用户想讨论线下零售策略

#### 第一轮：触发讨论
**用户**：我想讨论一下这个话题，并不是记录灵感。你能和我一起聊聊天吗？

**助理**：
```
那我问一个最核心的：

**你问这个问题，是纯好奇，还是跟你的工作/投资/创业有关？**

想听听你的背景，这样我们聊的时候能更有针对性～ 🦋

💬 83k/150k 55%
```

#### 第二轮：了解背景
**用户**：我是做 IT 产品渠道的，给代理商供货，想试试线下零售但效果不好。

**助理**：
```
明白了，你是从 B 端想扩展到 C 端。

**你心里想的线下店，主要是为了什么目的？**

- 直接卖给消费者赚差价？
- 给 B 端客户做展示和体验？
- 还是想测试新产品模式？

💬 89k/150k 59%
```

#### 第三轮：深入场景 + 引入关联
**用户**：主要是想给 B 端客户做个样板间，但发现光靠展示不够，还是得卖货。

**助理**：
```
我理解你的困境了——想做成"B 端展示 + C 端销售"的混合模式。

**你的下游代理商现在面临的最大困难是什么？**

（对了，我记得之前讨论过"C 端产品小 B 化"，你这个情况有点像反过来——"B 端渠道 C 端化"，你觉得呢？）

💬 97k/150k 65%
```

#### 第四轮：引入新视角
**用户**：代理商最大的问题是不知道卖什么，习惯了等客户下单。

**助理**：
```
好问题！之前的讨论核心是：个人用户也需要 B 端式的服务深度。

你现在的情况是：B 端代理商需要 C 端式的销售能力。

**你觉得线下店一定要靠卖货的差价来赚钱吗？**

有没有见过谁是用非差价的方式赚钱的？

💬 105k/150k 70%
```

#### 第五轮：挖掘隐性需求
**用户**：服务费...这个我确实没想过。但消费者会愿意为服务付钱吗？

**助理**：
```
**你买过什么产品，是买之前不知道需要，但用了之后觉得真香的？**

这种"真香"体验，往往就是隐性需求被满足了。

💬 112k/150k 75%
```

#### 第六轮：总结与下一步
**用户**：你这么一说，我想到几个点：代理商培训、售后支持、数据反馈。我回去整理一下，下次继续聊？

**助理**：
```
太好了！这三个方向都很有价值：
1. ✅ 代理商培训 → B 端能力提升
2. ✅ 售后支持打包 → 服务产品化
3. ✅ 数据反馈服务 → 数据价值变现

我已经把今天的讨论记录到 `memory/discussions/线下零售与 IT 产品策略_2026-03-01.md`

下次我们可以深入聊：这三种服务具体怎么定价？哪些代理商最需要？

随时找我继续！🦋

💬 119k/150k 79%
```

---

### 对话技巧总结

1. **每次只问 1-2 个问题** → 避免像审问
2. **根据用户回答调整** → 第三轮引入关联灵感
3. **适时引入新视角** → 第四轮提出"非差价赚钱"
4. **用具体例子引导** → 第五轮用"真香体验"类比
5. **及时总结与记录** → 第六轮整理要点 + 存储文件

## 📊 讨论质量评估

### 评估时机

**何时评估**：
- 讨论结束时（主动邀请用户评估）
- 讨论超过 10 轮时（中途检查质量）
- 用户表示"没收获"时（及时反思调整）

---

### 评估维度

| 维度 | 优秀 ✅ | 需改进 ❌ |
|------|--------|----------|
| **提问质量** | 每次 1-2 个问题，能引发深度思考 | 一次问太多/问题模糊 |
| **思维跳出** | 成功引入用户没想到的视角 | 只在用户框架内打转 |
| **关联召回** | 主动关联 2+ 个已有灵感 | 没有关联任何灵感 |
| **用户参与** | 用户主动延伸话题 | 用户表示疲惫/困惑 |
| **产出清晰** | 形成明确的待探索方向 | 讨论后仍很模糊 |
| **记录完整** | 每轮更新，关键洞察不遗漏 | 没有记录或很简略 |

---

### 评估方法

#### 方法 1：直接邀请用户反馈

**讨论结束时主动询问**：
```
今天的讨论对你有帮助吗？

- 有哪些问题让你觉得"这个角度我没想过"？
- 有哪些地方觉得我在绕圈子？
- 如果 1-10 分，你给今天的讨论打几分？

你的反馈能帮我下次聊得更好！🦋
```

#### 方法 2：自我检查清单

**每轮对话后快速自检**：
- [ ] 我这次只问了 1-2 个问题吗？
- [ ] 我的问题能帮用户跳出当前框架吗？
- [ ] 我有没有联想到相关的已有灵感？
- [ ] 用户的回答比上一轮更深入了吗？
- [ ] 我需要更新讨论记录文件吗？
